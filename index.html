From zero to shipped

🏗️

fledge init

Language-aware scaffolding. Templates with post-hooks, git init, and dependency setup. Zero config drift.

⚙️

fledge run

Auto-detects Cargo, npm, Go, Python. One command for build, test, lint — no Makefile required.

📐

fledge spec

Keep specs and code in sync. Drift detection fails CI before your API breaks production.

🤖

fledge review

Local AI code review via Ollama or Claude. No API keys for local models. Catches bugs, enforces conventions.

🚀

fledge release

Conventional commits → changelog → version bump → git tag → GitHub release. One command.

🧩

fledge plugin

Rust-based plugin system. Publish to crates.io, share across teams, compose custom workflows.

One binary. Three interfaces.

Built for teams, not just individuals

Standardized Config

One fledge.toml per repo. Every project runs the same way — no more “works on my machine.”

Spec Enforcement

Specs drift? CI catches it. fledge spec check ensures docs and code never diverge.

Fast Onboarding

fledge doctor validates the entire environment. fledge init scaffolds with team templates.

Code Metrics

LOC, test ratios, churn — fledge metrics gives leads visibility without extra tooling.

Extend without lock-in

Fledge plugins are Rust crates. Install from crates.io, git, or local paths.

# Install a community plugin
$ fledge plugin add coverage
# Use it immediately
$ fledge coverage report
# Or compose custom workflows
$ fledge lane run ci
